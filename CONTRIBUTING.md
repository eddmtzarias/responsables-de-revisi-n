# Normas de contribución

Gracias por interesarte en contribuir. Para colaborar sigue estas pautas:

1. Abre un issue describiendo el cambio que propones (si aplica).
2. Crea una rama con prefijo `feature/` o `fix/`.
3. Mantén commits pequeños y con mensajes claros.
4. Incluye o actualiza tests para cualquier funcionalidad nueva.
5. Abre un Pull Request hacia `main`. Asegúrate de que los checks de CI pasen.
6. Si el cambio implica decisiones de diseño o API públicas, documenta en el README o en docs/.

Responsables de revisión: revisiones requeridas por los codeowners configurados en `.github/CODEOWNERS`.
